# Container Agent - 产品需求文档

## Overview
- **Summary**: 快速搭建一套可直接运行、前后端数据库一体化的工作助手 Agent Web 应用基础骨架，实现核心权限管控与页面框架，预留后续 AI Agent 功能、业务工作模块的拓展空间。
- **Purpose**: 为企业级应用提供一个安全、可扩展的基础架构，减少从零搭建项目的时间和成本。
- **Target Users**: 企业内部用户，包括管理员和普通用户。

## Goals
- 实现全链路登录与角色权限管控
- 开发管理员专属的用户管理模块
- 构建系统首页与全局布局框架
- 提供完整的前后端分离架构，可直接运行

## Non-Goals (Out of Scope)
- 开发 AI Agent 对话功能
- 实现具体工作工具
- 开发复杂业务流程
- 实现数据统计、报表功能
- 实现文件上传下载功能
- 输出接口文档、详细设计文档

## Background & Context
- 企业级应用需要一个安全、可靠的基础架构
- 前后端分离架构已成为现代 web 应用的标准实践
- JWT 鉴权是目前主流的无状态认证方案
- Vue3 + Element Plus 提供了良好的前端开发体验
- Node.js + Express 是轻量级后端服务的理想选择
- MySQL 8.0 + Prisma ORM 提供了稳定的数据库解决方案

## Functional Requirements
- **FR-1**: 全链路登录与角色权限管控
  - 提供账号密码登录能力
  - 登录表单基础合规性校验
  - 登录异常明确提示
  - 双角色权限隔离体系（管理员、普通用户）
  - 全局登录状态管控
  - 安全退出能力
  - 系统初始化自动预置账号

- **FR-2**: 管理员专属：用户管理模块
  - 仅管理员可见、可操作
  - 用户全生命周期管理（新增、编辑、调整角色、启用/禁用、删除）
  - 可视化用户列表
  - 支持按账号/昵称关键词搜索筛选
  - 高危操作二次确认
  - 禁用状态账号禁止登录

- **FR-3**: 系统首页与全局布局框架
  - 登录成功后默认进入系统首页
  - 首页卡片式布局，展示功能入口
  - 入口卡片仅做UI占位与路由跳转预留
  - 统一后台管理经典布局（顶部导航栏 + 左侧菜单栏 + 主内容区）
  - 顶部导航栏展示用户信息和退出按钮
  - 左侧菜单栏动态权限渲染
  - 简约商务办公风UI

## Non-Functional Requirements
- **NFR-1**: 安全性
  - 用户密码不可逆加密存储
  - JWT 实现全链路登录鉴权
  - 权限严格管控

- **NFR-2**: 可扩展性
  - 前后端分离架构
  - 代码结构分层清晰、模块化拆分
  - 预留充足的拓展空间

- **NFR-3**: 可用性
  - 提供完整的环境配置说明
  - 依赖安装步骤清晰
  - 项目启动命令明确
  - 无报错、可正常访问

- **NFR-4**: 代码质量
  - 命名规范
  - 注释清晰
  - 符合企业级开发标准

## Constraints
- **Technical**: 
  - 前端：Vue3 + Vite + Element Plus + Axios + JWT
  - 后端：Node.js + Express
  - 数据库：MySQL 8.0 + Prisma ORM
  - 安全鉴权：JWT

- **Business**: 
  - 仅开发核心基础骨架功能
  - 禁止开发非核心业务功能

- **Dependencies**:
  - MySQL 8.0 数据库
  - Node.js 运行环境

## Assumptions
- 服务器环境已配置好 MySQL 8.0
- 开发环境已安装 Node.js
- 项目将部署在支持 Node.js 的服务器上

## Acceptance Criteria

### AC-1: 登录功能
- **Given**: 用户访问系统
- **When**: 用户输入正确的账号密码
- **Then**: 登录成功，跳转到系统首页
- **Verification**: `programmatic`

### AC-2: 权限管控
- **Given**: 普通用户登录系统
- **When**: 普通用户尝试访问用户管理模块
- **Then**: 无法看到用户管理菜单，直接访问URL会被拒绝
- **Verification**: `programmatic`

### AC-3: 用户管理
- **Given**: 管理员登录系统
- **When**: 管理员在用户管理模块执行操作
- **Then**: 可以成功新增、编辑、调整角色、启用/禁用、删除用户
- **Verification**: `programmatic`

### AC-4: 系统布局
- **Given**: 用户登录系统
- **When**: 用户浏览系统页面
- **Then**: 系统采用统一的后台管理经典布局，UI风格为简约商务办公风
- **Verification**: `human-judgment`

### AC-5: 项目运行
- **Given**: 开发者按照文档配置环境
- **When**: 开发者启动项目
- **Then**: 项目可以正常运行，无报错，全功能可正常使用
- **Verification**: `programmatic`
